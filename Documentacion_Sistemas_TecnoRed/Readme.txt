He empezado las practicas en el departamento de informatica de una empresa llamada TecnoRED SL, 
y mi mision es que organizar y documentar varios equipos y la red interna para tener todo el trabajo centralizado.
1. ¿Por qué la empresa usa Git y no carpetas normales?
Porque con carpetas normales se lía todo: se sobrescriben archivos, no sabes quién cambió qué y si algo se rompe, adiós al archivo. 
Git guarda todo el historial y deja que todos trabajemos al mismo tiempo sin perder nada.

2. ¿Qué ventaja tiene GitHub frente a un USB?
GitHub es en la nube, puedes trabajar desde cualquier sitio, colaborar con el equipo y ver todos los cambios.
 Un USB solo guarda la última versión y no sirve para trabajar juntos.
3. ¿Qué pasaría si alguien borra un archivo por error?
Con carpetas normales: se pierde.
Con Git lo puedes recuperar en segundos porque todo queda registrado en el historial.
4. ¿Por qué es importante escribir bien los commits?
Porque los commits son como notas del diario del proyecto.
